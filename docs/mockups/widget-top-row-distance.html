Widget v3.1 — пробег за поездку в верхней строке

Верхняя строка становится трёхслотовой: слева — время, в центре — пробег за поездку (новое), справа — T° салона.
Слоты равной ширины (CSS grid 1fr 1fr 1fr), внутри слотов выравнивание соответственно start / center / end.
Время переключается на компактный формат в часах: 25 мин / 1ч 25м — чтобы длинная поездка не налезала на центральный слот.

Изменения в FloatingWidgetView.kt:
1. RowTrip(sessionStartedAt, tripDistanceKm, insideTemp) — добавить новый параметр tripDistanceKm: Double?.
2. Внутри RowTrip: заменить Row(SpaceBetween) на Row с тремя Modifier.weight(1f)-колонками, каждая — Box(contentAlignment = Start / Center / End).
3. Центр: IconText(icon = Icons.Outlined.Route, text = formatTripKm(tripDistanceKm)).
4. formatTripKm: null → "—"; x < 10 → "%.1f км".format(x); x ≥ 10 → "%.0f км".format(x).
5. formatDurationShort: минуты — "$minutes мин"; часы — "${hours}ч ${minutes}м" (сокращённый формат).

Источник данных:
В TrackingService добавить val _tripDistanceKm = MutableStateFlow<Double?>(null) → public val tripDistanceKm: StateFlow<Double?>. Обновляется в polling loop из tripTracker.distanceKm (уже считается для BD). В WidgetController.attach() собирать этот Flow и передавать в FloatingWidgetView.
Если сессия закрыта (ignition-off): пробег null → виджет показывает —.

Иконка для пробега: Icons.Outlined.Route (Material Outlined) — две точки соединённые извилистой линией. Визуально не путается с Schedule (часы) и DirectionsCar (авто).

Формат времени — обоснование компактности:
Текущий "1 ч 25 мин" = 10 символов monospace 13sp ≈ 80dp. В трёхслотовой сетке слот 77dp — не влезет.
Компактный "1ч 25м" = 6 символов ≈ 48dp → запас 29dp на иконку и spacer.
В TrackingService уведомлении формат не трогаю — там места много.
