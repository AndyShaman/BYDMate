Как вернуть виджет

открыть BYDMate
• Тап по ярлыку BYDMate на главном экране DiLink
• Приложение открывается → виджет автоматически возвращается на своё место (позиция сохранена)
• Закрываешь BYDMate → виджет снова висит поверх
— никаких кнопок, никакого permission — просто открой приложение
Вёрстка виджета:
Размер: 260 × 108 dp, padding 8×14 dp, border 1.5 dp, corner 14 dp.
Строка 1 — ⏱ 22 мин · 🚗 25° (13sp моно). Иконки серые, значения белые.
Строка 2 (центр) — 78% 18sp bold (цвет по SOC) · ~352 км 28sp bold белый · ↓ 14.8 18sp (цвет по тренду).
Строка 3 — 🔋 28° · ⚡ 12.7 В. Тот же стиль что строка 1.
Разделители — 1dp line alpha 0.22.
Иконки: Material Outlined — Schedule, DirectionsCar, Battery6Bar, Bolt, TrendingDown/Flat/Up.

Поведение касаний:
• Обычный тап → открыть BYDMate (как сейчас).
• Drag → перемещение виджета; в зоне «корзина» (нижняя центральная) — отключение совсем (флаг в настройках).
• Long-press 1500 мс → скрыть до следующего открытия BYDMate. Подсветка во время нажатия, toast после срабатывания.

Техническая реализация long-press:
• В WidgetTouchListener добавить Handler с задержкой 1500 мс в ACTION_DOWN, отменять в ACTION_MOVE (если превысили threshold) и ACTION_UP.
• При срабатывании: WidgetPreferences.setHiddenUntilAppLaunch(true), WidgetController.detach(), Toast.makeText(...).
• В BYDMateApp.WidgetLifecycleCallbacks.onActivityResumed сбросить флаг false (виджет вернётся автоматически, когда Activity уйдёт в pause — стандартная логика attach в background).
• В WidgetController.attach добавить early-return если hiddenUntilAppLaunch == true.
